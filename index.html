Manual Retry

일반 코드: Retry 직접 조립

async function loadUser() {
  let lastError;
  for (let i = 0; i < 3; i++) {
    try {
      const res = await fetch("/api/user");
      if (res.ok) return await res.json();
    } catch (e) { lastError = e; }
    await sleep(1000 * (i + 1));
  }
  throw lastError;
}
loop · await · delay · error capture · 재시도 정책이 모두 섞여 있다.
27
Policy as Value

Retry도 값으로 조합한다

const loadUser = Effect.gen(function* () {
  const http = yield* HttpClient;
  return yield* http.getUser();
});

const resilient = loadUser.pipe(
  Effect.timeout("5 seconds"),
  Effect.retry(Schedule.exponential("1 second").pipe(
    Schedule.intersect(Schedule.recurs(2))
  ))
);
정책이 실행 코드에 섞이지 않고, 프로그램 값에 조합된다.

Generator

Generator = 멈출 수 있는 함수

function* program() {
  const a = yield "step 1";
  const b = yield "step 2";
  return a + b;
}

const gen = program();

console.log(gen.next().value); // "step 1"
console.log(gen.next(10).value); // "step 2"
console.log(gen.next(20).value); // 30
yield에서 멈추고, caller가 next()로 값을 넣어 resume.

Principles

Effect.ts가 가르쳐주는 6가지 원칙

  1. 순수 로직과 Effect 분리
  2. Effect를 값으로 다룸
  3. 성공 / 실패 / 의존성은 타입에
  4. 코드는 capability, runtime은 나중에
  5. 시간은 runtime이 제어
  6. generator로 program과 runtime 연결

Boundaries

Effect.ts가 아닌 것

  • 아키텍처를 자동으로 만들어주는 도구

CLOSING

명시적인 분리

Effect.ts는 아키텍처를 대신 만들지 않는다.
하지만 순수 로직/Effect, code/runtime, success/error, interface/implementation을
TypeScript 코드 안에서 명시적으로 표현하게 해준다.
Effect.ts가 가르쳐주는 것은 라이브러리 사용법이 아니라,
“무엇을 선언하고, 무엇을 runtime에게 맡길 것인가”를 다시 생각하는 것이다.
